# Task 15 – Vulnerability Assessment & Risk Prioritization

## Overview

This task demonstrates the process of performing a vulnerability assessment and prioritizing identified risks based on severity and potential impact. The objective is to analyze a target system, identify possible vulnerabilities, classify them according to risk level, and recommend remediation strategies.

## Tools Used

* Nmap (Network Scanner)
* Kali Linux
* CVE Reference Mapping
* CVSS Risk Scoring Model

## Target Scope

Target System: Localhost (127.0.0.1)
Environment: Kali Linux Virtual Machine
Assessment Type: Local Vulnerability Assessment

## Steps Performed

1. Defined the target scope and system for assessment.
2. Configured the vulnerability scanner with appropriate scan options.
3. Executed service detection and vulnerability scans using Nmap.
4. Reviewed scan results and analyzed potential vulnerabilities.
5. Demonstrated mapping of vulnerabilities to CVE and CVSS scores.
6. Classified vulnerabilities based on severity and potential impact.
7. Prioritized vulnerabilities according to risk level.
8. Proposed remediation strategies to mitigate identified risks.

## Key Findings

The vulnerability scan did not identify open ports or active services on the target system. This indicates a minimal attack surface and a relatively secure configuration.

However, the assessment demonstrated how vulnerabilities are typically mapped to CVE identifiers and evaluated using CVSS scoring for risk prioritization.

## Final Outcome

The task successfully demonstrates the methodology of vulnerability assessment, risk classification, and remediation planning within a cybersecurity framework.
